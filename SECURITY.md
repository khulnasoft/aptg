# Security Policy

## Supported Versions

Currently, the following versions of **aptg** are supported with security updates:

| Version | Supported |
| ------- | --------- |
| 0.1.x   | ✅        |
| < 0.1.0 | ❌        |

## Reporting a Vulnerability

We take the security of **aptg** seriously. If you discover a security vulnerability, please do NOT open a public issue. Instead, report it through the following channels:

1. **Email**: Send a detailed report to **security@khulnasoft.com**.
2. **Encrypted Communication**: If necessary, please ask for a GPG key to encrypt your report.

### What to include

- A description of the vulnerability.
- Proof of Concept (PoC) or steps to reproduce.
- Potential impact if exploited.

### Our Response Process

1. **Acknowledgement**: You will receive a response within 48 hours acknowledging your report.
2. **Investigation**: We will investigate and validate the vulnerability.
3. **Mitigation**: Once validated, we will work on a fix.
4. **Disclosure**: After a fix is released, we will provide credit (if desired) and publish a security advisory.

Thank you for helping keep the Debian ecosystem secure!
